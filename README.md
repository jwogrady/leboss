# LEBOSS
## Local Entrepreneur Business Operating System Standards

**Current Version:** 0.0.11 (Draft) — preparing for 0.1.0 implementable draft release

---

LEBOSS is an open standard. Its core position is simple: **the data generated by a local business belongs to the owner of that business** — not the software company, not the cloud platform, not the agency. This standard makes that ownership architectural — built into the structure of systems — rather than merely contractual.

## What the Standard Defines

**Five Foundation Principles:**
1. Clarity — every element has a defined role and place
2. Modularity — capabilities are interchangeable, not entangled
3. Security — data separated by entity, access is least-privilege, all actions auditable
4. Legacy and Continuity — systems survive ownership transitions
5. Extensibility — new capabilities attach without disrupting existing ones

**The LEBOSS Reference Model — six hierarchical elements:**

| Element | Name | Role |
|---------|------|------|
| 0 | Universe | Root owner — the governing entity that owns everything |
| 1 | Galaxy | A brand or business line owned by the Universe |
| 2 | Star | The customer experience — websites, apps, portals |
| 3 | Planet | The backend service powering a Star |
| 4 | Moon | Company-owned internal capabilities (natural satellites) |
| 5 | Satellite | Third-party integrations (artificial satellites) |

**Data Ownership Doctrine:** Primary operational data belongs to the governing entity. Service providers are stewards, not owners. Access is explicitly granted, scoped, and auditable.

## Presentation System

The LEBOSS repository includes a multi-deck Slidev presentation system organized by audience. All decks are in `presentations/` and share a common Slidev build environment.

| Deck | Path | Audience |
|------|------|----------|
| **Overview** | [presentations/slidev/overview.md](presentations/slidev/overview.md) | Business owners, solution providers, developers evaluating the ecosystem |
| **Architecture** | [presentations/slidev/architecture.md](presentations/slidev/architecture.md) | Developers, platform architects, technical implementers |
| **Governance** | [presentations/slidev/governance.md](presentations/slidev/governance.md) | Contributors, maintainers, committee members |

The Overview deck is deployed to **[leboss.status26.com](https://leboss.status26.com/)** via Netlify.

To run any deck locally:

```bash
cd presentations/slidev
npm install
npm run dev                  # overview (default)
npm run build:architecture   # architecture deck
npm run build:governance     # governance deck
```

The original presentation — the historical source from which the formal standard was derived — is preserved as an archive artifact at [presentation/slides.md](presentation/slides.md) (singular directory).

## Why This Exists

Local businesses — plumbers, dentists, restaurants, landscapers, retailers — have adopted digital tools at scale. In doing so, most have handed operational control of their data to software vendors who designed their platforms to make switching expensive and data portability minimal.

In the AI era, this problem is more acute. AI systems are trained on business data. Insights derived from local business operations are monetized by platforms. The business owner generated the data. The platform captures the value.

LEBOSS proposes that this is a design choice, not an inevitability. And that developers who build better can change it.
